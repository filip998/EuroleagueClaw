## Mission

Investigate whether EuroLeague PlayByPlay supports incremental fetching to reduce bandwidth consumption during live game polling.

## Key Findings

1. **Zero Incremental API Support**
   - Tested 8 different query parameter styles and conditional headers
   - All returned identical full 157 KB payload
   - No server-side filtering, etags, or `since` parameter support

2. **Gzip Compression Active** ✅
   - Wire transfer: ~10.7 KB (6.8% of uncompressed)
   - Already highly optimized at transport layer

3. **Lightweight Alternative Endpoints Discovered**
   - `/api/Header`: 475 bytes gzipped (has live scores and clock)
   - `/api/Points`: 4.5 KB gzipped (scoring plays only)
   - Could enable "poll Header first, fetch PBP only on score change" optimization if needed

## Recommendations

**Immediate:** Full PBP fetch is acceptable at current polling frequency (15s).  
**Future:** If bandwidth becomes constraint, use Header/Points endpoints for lightweight polling before full PBP fetch.
