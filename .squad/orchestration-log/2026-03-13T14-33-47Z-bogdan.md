## Critical Finding

**PBP is only used for roster matching.** Score detection, quarter transitions, lead changes, and big runs all use `getLiveScore()`, not PBP.

## Payload Analysis

- Full game PBP: ~154 KB / 578 events
- Notable events (cared about): 27%
- Client-side filtering via `sinceEventId` after full fetch
- At 15s polling over 2-hour game: ~45 MB of mostly redundant transfers per tracked game

## Two "Free Wins" (No API Probing Required)

1. **Skip PBP fetch when rosters aren't loaded**
   - Currently: Fetch 154 KB → callback returns early (no roster match)
   - Fix: Guard `pollingCallback` with roster presence check

2. **Reduce PBP poll frequency to 30–45s**
   - Roster notifications less time-critical than score updates
   - Still responsive enough for fantasy tracking

## Optimization Strategy (Ranked)

1. **Skip fetch when no rosters** — 0 code complexity
2. **Reduce poll frequency** — Trivial change
3. **Use lightweight endpoints first** (pending Nikola's API probe) — if Header/Points available
4. **Implement quarter filtering** (if API supports) — medium complexity
